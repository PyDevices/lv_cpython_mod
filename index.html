lv_cpython_mod

Native CPython extension for LVGL — import lvgl, with no MicroPython runtime. Generated C comes from lv_bindings (generated/lvgl_python.c) and is auto-synced on push to main. Prebuilt wheels ship as lvgl-cpython on TestPyPI for CPython 3.10–3.14 (Linux x86_64, Windows x64) plus Android wheels for 3.13–3.14.

Use it when you want LVGL widgets beside pydisplay on the host, in CI, or inside an APK via pydisplay_android.

TestPyPI GitHub lv_bindings

Install from TestPyPI

Import as lvgl; package name is lvgl-cpython 
pip install -i https://test.pypi.org/simple/ \
  --extra-index-url https://pypi.org/simple/ lvgl-cpython

python -c "import lvgl as lv; lv.init(); lv.deinit(); print('ok')"

Usage excerpt

Hello label

Display setup uses module-level functions, not methods on the display object.

import lvgl as lv

lv.init()
disp = lv.display_create(240, 240)
lv.display_set_color_format(disp, lv.COLOR_FORMAT.RGB565)
buf = lv.draw_buf_create(240, 240, lv.COLOR_FORMAT.RGB565, 0)
lv.display_set_draw_buffers(disp, buf, None)
lv.display_set_render_mode(disp, lv.DISPLAY_RENDER_MODE.PARTIAL)

scr = lv.screen_active()
label = lv.label(scr)
label.set_text("Hello")
lv.deinit()

Android wheel

Install a platform wheel when building an APK, or let the lvglcpython p4a recipe fetch it.

pip install -i https://test.pypi.org/simple/ \
  --extra-index-url https://pypi.org/simple/ \
  --only-binary=:all: \
  --platform android_21_arm64_v8a \
  lvgl-cpython

Pyodide / WebAssembly

Native TestPyPI wheels do not run in Pyodide. Build a pyemscripten_2026_0 wheel with ./scripts/build_pyodide_wheel.sh; artifacts land in wheels/. See BUILDING.md.
